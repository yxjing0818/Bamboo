# AI 辅助几何算法开发：12周进阶系统学习大纲

这份大纲专为**复杂曲面方向的算法工程师**设计，摒弃了泛泛的 AI 理论，直接切入核心痛点：公式推导、边界退化处理、C++工程落地、以及与 OpenCASCADE/Eigen 等库的深度融合。

每三到四周为一个阶段，确保系统性与实操性。

---

## 阶段一：重塑人机交互——从“自然语言”到“数学化/约束化提示词”（第 1-4 周）

**核心目标：** 抛弃“帮我写个XX算法”的菜鸟问法，学会用 LaTeX、伪代码和边界约束来精确控制 AI 的代码生成，重点解决几何算法中的“退化（Degenerate）”难题。

### 第 1 周：基础与表达（焦点：曲线求值与公式化 Prompt）
*   **学习重点**：理解 LLM 对数学符号的解析逻辑，掌握“公式描述 -> 伪代码 -> C++ 代码”的三步走 Prompt 模板。
*   **周一至周二**：**对比训练**。
    *   *任务*：实现一个 Bezier 曲线求值器。分别尝试用大白话和 LaTeX 数学公式（如 De Casteljau 算法）向 Gemini 提问，观察代码健壮性的差异。
*   **周三至周四**：**递归与复杂公式的 Chain-of-Thought (思维链)**。
    *   *任务*：实现 B 样条基函数求值（Cox-de Boor 递推）。
    *   *Prompt 技巧*：不要直接让它写代码，要求它先推导 $N_{i,3}(u)$ 的展开式，确认分母不为零的条件，再进行代码生成。
*   **周五至周日**：**NURBS 与奇点处理**。
    *   *任务*：NURBS 曲线求值。
    *   *Prompt 技巧*：在 Prompt 中明确规定：“请特别注意权重 $w=0$ 或节点重复度（Knot Multiplicity）等于阶数时的除零风险，并给出相应的 `epsilon` 容差处理方案。”

### 第 2 周：数值优化与边界约束（焦点：求交、投影与退化情况）
*   **学习重点**：训练 AI 写出能应对恶劣输入（如三点共线、极小面积、重合点）的防御性代码。
*   **周一至周二**：**基于 Newton-Raphson 的点到曲线投影（Point Inversion）**。
    *   *任务*：给定空间一点求在 B 样条曲线上最近点的参数 $(u, v)$。
    *   *Prompt 技巧*：向 AI 描述雅可比矩阵，并要求其实现在迭代发散或步长过小（`Step < 1e-6`）时的回退机制（Fallback to bounding box subdivision）。
*   **周三至周四**：**射线投射与点在多边形内判定（Point-in-Polygon）**。
    *   *任务*：经典的射线法判断点是否在任意多边形内。
    *   *Prompt 技巧*：在提示词中强制列出测试用例：“你的代码必须能正确处理射线刚好穿过多边形顶点、或与多边形边共线的情况。”
*   **周五至周日**：**三角网格基础相交测试（Triangle-Triangle Intersection）**。
    *   *任务*：实现快速且鲁棒的三角形相交测试。
    *   *Prompt 技巧*：要求 AI 使用基于行列式符号判定（Orientation Predicates）的算法（如 Moeller-Trumbore），而不是直接计算交点坐标以避免精度丢失。

### 第 3 周：上下文注入 (In-Context Learning)（焦点：融入现有 C++ 架构）
*   **学习重点**：教 AI 使用你的自定义数据结构和第三方核心库，而不是让它从头造轮子。
*   **周一至周二**：**自定义数据结构注入**。
    *   *任务*：提取你的项目代码中的 `Point3D.h` 和 `Vector3D.h` 文件内容。
    *   *Prompt 技巧*：将头文件代码作为上下文发给 Gemini，提问：“请严格且仅使用上述头文件中的类及其操作符重载，实现一个计算空间多边形面积的函数。禁止使用 `std::vector` 等 STL 容器。”
*   **周三至周四**：**Eigen 库的深度调用**。
    *   *任务*：基于点云数据的最小二乘平面的拟合。
    *   *Prompt 技巧*：明确指定库和优化目标：“请使用 C++ 和 Eigen 库实现 SVD 分解求法向量，要求避免多余的内存拷贝，直接使用 `Eigen::Map` 处理原始浮点指针数据。”
*   **周五至周日**：**内存与性能审查 (Code Review)**。
    *   *任务*：把你写的一段复杂的嵌套循环（如计算网格高斯曲率）喂给 AI。提问：“请以 C++ 性能专家的身份 Review 这段代码，指出其中是否存在多余的堆内存分配或缓存未命中（Cache Miss）的问题。”

### 第 4 周：第一阶段综合项目实战
*   **考核项目**：**带尖锐特征保留的网格平滑（Feature-preserving Mesh Smoothing）**。
*   **执行流**：
    1. 用 LaTeX 给出双边滤波（Bilateral Filtering）在法向量上的数学公式。
    2. 将项目的基础网格半边数据结构（Half-edge data structure）头文件发给 AI。
    3. 让 AI 分步写出：计算每个面法向 -> 邻域权重计算 -> 顶点位置更新 的 C++ 代码。
    4. 对照 AI 的代码，检查其是否正确处理了开放边界（Open Boundaries）。

## 阶段二：数学引擎与自动微分——打通推导到落地的闭环（第 5-8 周）

**核心目标：** 将 AI 提升为你的“数学推导搭档”。利用 Python(SymPy) + AI 解决曲面变形、优化求导等人工极难保证绝对正确的数学难题，并自动化生成 C++ 代码。

### 第 5 周：符号推导基础与 SymPy 结合
*   **周一至周三**：**告别手算偏导数**。
    *   *任务*：学习如何让 Gemini 写 Python 脚本来做数学推导。
    *   *练习*：推导一个特定非理性 B 样条曲面在某个控制点发生位移时的能量变化率。让 Gemini 生成 SymPy 脚本，自动求出 Gradient 解析式。
*   **周四至周日**：**复杂能量泛函的 Hessian 矩阵生成**。
    *   *任务*：在曲面拟合（Surface Fitting）中，为了使用 L-BFGS 等优化器，需要计算黑塞矩阵。
    *   *练习*：给 AI 泛函的积分定义，让它不仅推导出 Hessian 的解析式，还要让它输出为 C/C++ 友好的代码格式（`ccode` 或 `cxxcode` 格式）。

### 第 6 周：从符号解析解到工业级 C++ 代码
*   **周一至周三**：**公共子表达式消除（CSE, Common Subexpression Elimination）**。
    *   *任务*：SymPy 导出的解析式往往极其冗长，计算效率极低。
    *   *练习*：把上周生成的几百行长公式喂给 AI，要求：“请在将其转为 C++ 代码时，手动提取公共子表达式，以减少 CPU 的乘加运算（FMA）次数。”
*   **周四至周日**：**与 Eigen 稀疏矩阵的完美拼接**。
    *   *任务*：将局部偏导数高效组装到全局稀疏矩阵中。
    *   *练习*：给 AI 提供局部能量计算函数，提问：“我有 $N$ 个顶点，请用 C++ 给出如何利用 `Eigen::Triplet` 将上述局部 Hessian 组装为全局稀疏矩阵的最高效写法。”

### 第 7 周：用大模型征服重型几何开源内核
*   **周一至周三**：**OpenCASCADE (OCCT) 逆向探索**。
    *   *任务*：OCCT 文档极少。
    *   *练习*：向 Gemini 提问：“我需要用 OpenCASCADE 实现一个复杂的布尔差集（Boolean Cut），但保留特定的相交线。请给我使用 `BRepAlgoAPI_Cut` 或底层 `BOPAlgo_Builder` 的代码范例，并解释它们在处理容差（Tolerance）上的区别。”
*   **周四至周日**：**CGAL 概念体系的 AI 解析**。
    *   *任务*：CGAL 深度依赖 C++ 模板编程，报错信息如同天书。
    *   *练习*：遇到 CGAL 模板编译报错时，将一整页的报错信息和你的代码发给 Gemini，提问：“请帮我解析这个 CGAL 模板编译错误，是不是我的 Kernel 选择（Exact_predicates_inexact_constructions）不匹配这个特定的细分表面算法？”

### 第 8 周：第二阶段综合项目实战
*   **考核项目**：**基于 ARAP (As-Rigid-As-Possible) 的曲面变形核心算子**。
*   **执行流**：
    1. 用 Gemini 解释 ARAP 论文中局部旋转矩阵 $R_i$ 和协方差矩阵 $S_i$ 的 SVD 分解数学过程。
    2. 引导它写出基于 Eigen 的 C++ 代码。
    3. 特别要求它处理当 $S_i$ 行列式为负时，SVD 分解的符号反转问题（退化处理）。

## 阶段三：AI 前沿应用探索与工作流定制（第 9-12 周）

**核心目标：** 超越传统编程，探索 AI 在几何自动化（Generative CAD）、代码逆向工程及系统性 Bug 定位上的高级应用，建立你个人的智能知识库。

### 第 9 周：深水区 Debug 与数值浮点灾难排查
*   **周一至周三**：**浮点数灾难 (Floating-Point Catastrophe) 定位**。
    *   *任务*：两个靠得很近的超大浮点数相减会导致有效数字丢失。
    *   *练习*：找一段计算曲率或非常细微几何特征的代码发给 AI：“请运用数值分析理论审查这段代码，找出所有可能发生『大数吞小数』或『相近数相减』导致精度丢失的地方，并用 Kahan 求和或重写公式来改进。”
*   **周四至周日**：**内存泄漏与指针审查**。
    *   *任务*：用 AI 排查 C++ 的底层问题。如果涉及 OpenCASCADE，其基于引用计数的 `Handle` 机制常常引发问题。
    *   *练习*：提问：“在这段基于 OpenCASCADE `TopoDS_Shape` 的迭代代码中，是否存在未正确解绑的 `Handle` 导致内存无法释放？请给出改进后的 RAII 写法。”

### 第 10 周：几何遗留代码的 AI 逆向工程
*   **周一至周四**：**从代码还原数学原理**。
    *   *任务*：工业软件中常有无人维护的祖传代码。
    *   *练习*：选取一段无注释的遗留算法（如一段旧的网格剖分逻辑），要求 Gemini：“1. 用伪代码和 LaTeX 还原它最初设计的数学原理；2. 指出其局限性；3. 给出等价的现代 C++ (C++11/14) 重构版本。”
*   **周五至周日**：**自动生成几何测试用例（Unit Testing）**。
    *   *任务*：让 AI 帮你写测试。
    *   *练习*：将你的求交函数发给 AI：“请基于 Google Test 框架，为这个面面求交函数编写不少于 5 个测试用例，必须包含极端情况（如完全重合、只有一个交点、T型相交）。”

### 第 11 周：走向 AIGC——几何大模型与参数化生成初探
*   **周一至周四**：**代码即 CAD (Code-as-CAD)**。
    *   *任务*：理解自然语言如何直接驱动 3D 建模。
    *   *练习*：尝试使用 `llmcad` 开源项目或者 FreeCAD 的 Python API。通过 Python 脚本调用 Gemini 的 API，尝试实现：输入自然语言“创建一个带 4 个螺栓孔的法兰盘”，输出完整的 Python OCC 或 FreeCAD 建模代码，并自动运行生成 `.step` 文件。
*   **周五至周日**：**论文深度泛读**。
    *   *练习*：下载《DeepCAD》或《CAD-GPT》等前沿学术论文。要求 Gemini 帮你做学术剥析：“这篇论文是如何将三维的 BREP 拓扑结构序列化为 LLM 能理解的 1D Token 序列的？请用通俗的几何语言向我解释。”

### 第 12 周：制定你的专属 AI 辅助开发 SOP (标准作业程序)
*   **核心任务**：沉淀 12 周的知识，形成你个人的工作宝典。
*   **具体产出要求**：
    1.  **建立个人 Prompt 词典**：整理出你在推导公式、处理 OpenCASCADE 报错、进行 Eigen 优化时最有效的 5-10 个 Prompt 模板。
    2.  **构建基础上下文库 (Context Library)**：将项目中最重要的十几个核心 `.h` 结构体打包成一个独立的文本文件。以后每次需要 AI 写代码时，先把它作为前置上下文发给 AI。
    3.  **分享与复盘**：思考在这个流程中，AI 目前绝对不能替代的部分（通常是底层几何数据架构设计和极端的容差调试），将时间精力进一步倾斜到这些不可替代的高价值工作上。
